# OSSライブラリ調査メモ

## 推薦候補

| ライブラリ | 用途 | 導入判断 |
|---|---|---|
| yfinance | Yahoo Finance由来の市場データ取得 | 価格・銘柄・履歴データを扱うrepo向け |
| ta | Pandas/Numpyベースのテクニカル指標 | OHLCV、RSI、MACDなどの特徴量生成向け |
| backtesting.py | 軽量バックテスト | 戦略・売買シグナルrepo向け |
| quantstats | 成績分析・HTMLレポート | リターン系列やバックテスト結果を持つrepo向け |
| PyPortfolioOpt | 古典的ポートフォリオ最適化 | 平均分散、Black-Litterman、リスクモデル向け |
| skfolio | ML親和性の高いポートフォリオ最適化 | scikit-learn風の検証・最適化パイプライン向け |
| OpenBB | 多ソース金融データ基盤 | ファンダメンタル、ニュース、複数データプロバイダ統合向け |

## 反映ポリシー

- 既存コードへの直接変更は避ける
- まず改善提案ドキュメントと候補requirementsをPR化
- 本番依存へ入れるかは対象repoのCI、利用規約、データソース方針を確認してから判断
- 金融データや売買判断ではなく、開発基盤の品質向上に限定
